# 테스트 설계 문서 — 나는 어떤 간식?

> 참고: 앱 폴더명은 초기 주제(conflict-style)를 따르지만, 사용자 요청으로
> 주제를 **"나는 어떤 간식?"** 으로 전면 변경했다(2026-07-22).

## 1. 테스트 이름

**나는 어떤 간식?** (달콤 취향 간식 테스트, 슬러그: `snack-type-test`)

## 2. 테스트 설계 유형

**B. 엔터테인먼트 상황 테스트**

- 검증된 심리척도를 사용하지 않는다.
- 3개의 독립적인 행동·취향 축을 정의하고, 각 응답이 결과(간식)가 아니라
  축 점수에만 영향을 주도록 설계했다.
- 계산된 3축 프로필을 5개 간식의 기준 프로필과 거리 비교하여 최종 결과를 정한다.

## 3. 테스트 목적

요즘 취향과 기분을 가볍게 돌아보며, 나와 닮은 간식을 재미로 찾아보고
결과를 이미지로 저장·공유하도록 돕는다.

## 4. 대상 사용자

- 20~40대, 모바일에서 2분 내외로 가볍게 즐기고 결과를 공유하고 싶은 사용자

## 5. 응답 기준 상황

**"요즘 나의 취향과 기분"** 을 떠올리며 응답한다.

## 6. 측정하는 취향 축 (3개)

| 축 ID | 이름 | 정의 |
| --- | --- | --- |
| `rich` | 진함 | 진하고 강렬한 맛·취향에 끌리는 정도 |
| `playful` | 발랄함 | 톡톡 튀고 활발한 재미를 즐기는 정도 |
| `warm` | 다정함 | 부드럽고 포근하게 곁을 챙기는 정도 |

세 축은 서로 독립적이며, 세 축의 조합으로 결과 간식을 규정한다.

## 7. 문항 구성

- **전체 문항 수: 7문항**
- **응답 방식: 문항당 4개 선택지 중 1개 선택** (상황·취향형)
- 각 문항 ID: `q1`~`q7` (중복 없음)
- 각 선택지는 3개 축에 0~2점을 가산한다(`scores`). **선택지가 특정 간식을
  직접 가산하지 않는다** — 오직 축 점수에만 영향을 준다.

## 8. 점수 계산 방식

### 8-1. 응답값 변환·합산
모든 문항에 응답이 있어야 계산한다(미응답 시 결과 없음). 선택한 선택지의
`scores`를 축별로 합산한다.

### 8-2. 정규화 (축별 최소~최대)
각 축의 이론적 최소/최대 원점수를 문항별 선택지의 min/max 합으로 구한 뒤,
`정규화 = 1 + 4 × (원점수 - 최소) / (최대 - 최소)` 로 1~5로 변환한다.
현재 데이터 기준 최소=`[0,0,0]`, 최대=`rich 14 / playful 14 / warm 14`.
규준 데이터가 없으므로 백분위로 해석하지 않는다("상위 %" 등 미사용).

### 8-3. 결과 프로필 비교
3축 좌표 `(진함, 발랄, 다정)` 를 5개 간식의 기준 프로필과 **가중 없는
유클리드 거리**로 비교한다. 최소 거리 = 대표, 2번째 = 보조.

동점 처리: 거리 차 1e-9 이내면 고정 순서(초콜릿→사탕→비스킷→마시멜로우
→푸딩)를 따른다. **무작위 미사용, 동일 응답 = 동일 결과.**

## 9. 결과 유형 (5개 간식)

| 간식 | 기준 프로필 (진함, 발랄, 다정) | 성격 |
| --- | --- | --- |
| 초콜릿 | (4.3, 2.5, 2.0) | 진하고 강렬, 확실한 취향 |
| 사탕 | (2.1, 4.4, 2.6) | 톡톡 튀는 발랄 |
| 비스킷 | (3.3, 2.0, 2.9) | 담백하고 든든, 한결같음 |
| 마시멜로우 | (1.9, 3.3, 3.8) | 발랄 + 다정, 사랑스러움 |
| 푸딩 | (2.6, 1.7, 4.4) | 포근한 다정 |

결과 수를 5개로 한 이유: 세 축의 대표 방향(진함/발랄/다정)과 그 사이 조합을
고르게 덮으면서, 각 간식이 서로 구분되고 모두 도달 가능하기 때문이다.
프로필은 실제 선택지로 도달 가능한 취향 공간 안에 배치했다(도달성 시뮬레이션으로
5개 모두 대표 결과로 등장함을 확인).

각 간식에는 `src/data/results.ts`에 이름, 한 줄 요약, 기준 프로필, 상세 설명,
매력(강점), 주의할 점, 팁, 보조 결과와의 관계, 공유 문구, 색상을 정의한다.
일러스트는 캔버스로 렌더링한 **PNG 이미지 파일**(`src/assets/snacks/*.png`)을
`src/components/SnackImage.tsx`로 표시하고, 저장 이미지(canvas, `lib/share.ts`)에도
같은 PNG를 불러와 합성한다. (PNG 자산은 `@napi-rs/canvas` 기반 생성 스크립트로
제작. 앱 런타임 의존성에는 포함하지 않음.)

## 10. 결과가 나온 이유 설명 방식

결과 화면에서 가장 높은 축과 두 번째 축의 점수를 문장으로 안내하고, 그 취향
조합이 대표 간식과 가장 가까웠다는 점을 설명한다. 사용자를 단정하지 않고
"현재 응답에서는 이 간식과 가장 가까웠어요" 표현을 사용하며, 보조 간식도
함께 보여준다.

## 11. 테스트의 한계

- 검증된 심리척도를 사용하지 않은 엔터테인먼트 콘텐츠다.
- 7문항 자기보고식이며 상황·기분에 따라 결과가 달라질 수 있다.
- 전문 심리 진단·성격검사·인사평가 용도가 아니다.
- 실제 사용자 파일럿 검증은 실시하지 않았다.

## 12. 참고 문헌 / 창작 구분

- **참고한 검증 척도: 없음** (엔터테인먼트 콘텐츠)
- 3개 취향 축, 12개 문항·선택지, 5개 간식 결과 및 일러스트는 모두 **새로 창작**했다.
- 특정 캐릭터·저작물을 복제하지 않았다.

## 13. 실제 사용자 파일럿 검증

- 테스트 성격: 엔터테인먼트
- 실제 사용자 검증: 미실시
- 도달성/결정성: 전체 응답 조합(4^7 = 16,384가지) 브루트포스 검사로 5개 결과가
  모두 대표 결과로 도달 가능함을 확인했고, 동일 응답에 동일 결과가 나온다.
